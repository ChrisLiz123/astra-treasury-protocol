# Restricted-Mode Operator Checklist

## Purpose

Give operators a practical checklist for steady-state restricted Base Mainnet operation.

## Current approved mode

Restricted Base Mainnet operation.

## Governance state

Governance decision is recorded for restricted-mode / all-disabled posture.

## Approved capabilities

None.

## Disabled / not approved

- Full launch.
- Public token sale.
- Real treasury funding.
- Staking or rewards.
- Buyback program.
- Mainnet execution queue activation.
- Paper-to-on-chain automation.
- Autonomous execution.
- Safe transaction payload generation.
- Safe transaction execution.

## Operator rule

Operators must not treat restricted-mode release, maintenance, handoff, monitoring, or evidence artifacts as capability activation approval.
