# Capability Activation Intake Checklist

## Before accepting a request

- [ ] Exact capability is named.
- [ ] Capability is currently disabled.
- [ ] Requester reference is present.
- [ ] Governance authority reference is present.
- [ ] Evidence reference is present.
- [ ] Public status plan is present.
- [ ] On-chain impact is assessed.
- [ ] Safe transaction impact is assessed.
- [ ] Mainnet monitor is passing.
- [ ] Active incidents are zero.
- [ ] Execution queue is disabled.
- [ ] Treasury funding remains not approved.
- [ ] Safe payload remains not generated.

## Rule

An accepted intake request is not capability approval.
