# Capability Activation Intake Gate

## Purpose

Create a controlled intake process for any future capability activation request.

## Current approved mode

Restricted Base Mainnet operation.

## Governance state

Governance decision is recorded for restricted-mode / all-disabled posture.

## Approved capabilities

None.

## Disabled / not approved

- Full launch.
- Public token sale.
- Real treasury funding.
- Staking or rewards.
- Buyback program.
- Mainnet execution queue activation.
- Paper-to-on-chain automation.
- Autonomous execution.
- Safe transaction payload generation.
- Safe transaction execution.

## Intake rule

The intake gate only accepts a future request for review. It does not approve, activate, fund, generate Safe payloads, or enable execution.

## Required for a future request

- Exact capability name.
- Requester reference.
- Governance authority reference.
- Purpose and scope.
- Evidence reference.
- Public status update plan.
- On-chain impact assessment.
- Safe transaction impact assessment.
- Separate action-specific activation path.
