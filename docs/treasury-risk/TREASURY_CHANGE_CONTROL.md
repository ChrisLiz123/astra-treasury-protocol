# Treasury Change Control

## Purpose

Define how treasury risk limits can change.

## Required process

1. Draft the proposed change.
2. Review legal/accounting implications.
3. Review asset and counterparty risks.
4. Review monitoring requirements.
5. Update public disclosures if needed.
6. Record governance approval.
7. Record Treasury Safe approval.
8. Update risk decision record.
9. Generate evidence snapshot.

## Emergency change process

Emergency changes must be documented after the fact with:

- reason for change
- approver
- timestamp
- transaction hash, if any
- impact assessment
- follow-up action

## Rule

Risk limits cannot be casually changed by editing a config file.
