# Treasury Safe Transaction Confidentiality

## Do not publish

- Private signer details.
- Private signing procedures.
- Private wallet-control information.
- Bank account details.
- Sensitive source-of-funds details.
- Private compliance notes.
- Non-public counterparty details.
- Pre-release Safe payloads that have not been approved for publication.

## Public-safe information

- Whether a package exists.
- Whether a payload has been generated.
- Whether a Safe transaction has been submitted.
- Whether a transaction has been executed.
- Whether funding is authorized.
- Whether funding is executed.
- Public transaction hash only after execution, if applicable.

## Rule

Public pages should expose only sanitized transaction-package status.
