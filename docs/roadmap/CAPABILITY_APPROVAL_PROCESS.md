# Capability Approval Process

## Purpose

Define how future AstraTreasury capabilities can be reviewed without accidentally enabling them.

## Required process for every capability

1. Define capability scope.
2. Identify legal implications.
3. Identify security implications.
4. Identify treasury or operational risks.
5. Create a dry-run or simulation where applicable.
6. Create public disclosure draft.
7. Get governance approval.
8. Update public status pages.
9. Enable only through a separate controlled milestone.

## Required evidence

- Approval record.
- Legal/security notes.
- Dry-run report where applicable.
- Updated public docs.
- Updated evidence archive.
- Monitoring plan.

## Rule

No capability can be enabled by a single operator casually changing a config flag.
