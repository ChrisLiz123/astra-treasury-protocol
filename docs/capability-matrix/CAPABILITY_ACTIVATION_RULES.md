# Capability Activation Rules

## Rule 1

A capability is disabled unless explicitly approved.

## Rule 2

A governance decision must name the exact capability being approved.

## Rule 3

A generic approval does not activate a capability.

## Rule 4

A Safe transaction must not be generated unless the relevant capability and transaction path are approved.

## Rule 5

Public status must be updated before and after any capability activation milestone.

## Rule 6

Evidence must be archived for every approval and activation step.
