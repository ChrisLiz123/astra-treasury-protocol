# Capability Matrix

## Purpose

Finalize the capability approval matrix for AstraTreasury.

## Current approved mode

Restricted Base Mainnet operation.

## Matrix conclusion

The capability matrix is finalized with every restricted capability disabled and not approved.

## Finalized disabled capabilities

- Full launch: not approved.
- Public token sale: not approved.
- Real treasury funding: not approved.
- Staking or rewards: not approved.
- Buyback program: not approved.
- Mainnet execution queue: not approved.
- Paper-to-on-chain automation: not approved.
- Autonomous execution: not approved.
- Safe transaction payload generation: not approved.
- Safe transaction execution: not approved.

## Meaning of finalization

This finalizes the current capability-state matrix. It does not approve or enable any capability.

## Rule

Every future capability requires a separate action-specific approval and evidence trail.
