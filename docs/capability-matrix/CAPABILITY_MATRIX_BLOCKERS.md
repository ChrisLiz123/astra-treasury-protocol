# Capability Matrix Blockers

The matrix itself has no blocker because it is finalized as all-disabled.

Capability activation remains blocked by default.

## Activation blockers

Each future capability requires:

- Explicit action-specific approval.
- Governance decision or capability-specific decision record.
- Evidence reference.
- Public status update.
- Monitoring confirmation.
- Incident-status confirmation.
- Safe review if on-chain execution is involved.

## Rule

A finalized matrix is not a capability activation.
