# Post-Announcement Monitoring

## Purpose

Define what to check after publishing the restricted launch announcement.

## Immediately after posting

- [ ] Check announcement URL.
- [ ] Check all linked public pages.
- [ ] Check mainnet monitor.
- [ ] Check event monitor.
- [ ] Check alerts.
- [ ] Check incidents.
- [ ] Check restricted launch status.
- [ ] Capture evidence archive snapshot.
- [ ] Confirm no active high/critical alerts.

## First 24 hours

- [ ] Review public links periodically.
- [ ] Watch operator notifications.
- [ ] Watch event monitor.
- [ ] Watch incident tracker.
- [ ] Preserve any notable evidence snapshots.

## Rule

If an alert or incident appears after announcement, acknowledge it through the private incident workflow before making additional public claims.
