# Capability Unlock Checklist

## Purpose

Define what must happen before any restricted mainnet capability is enabled.

## Public token sale

- [ ] Separate legal approval.
- [ ] Securities-law review updated.
- [ ] Public disclosures updated.
- [ ] Governance approval recorded.
- [ ] Website language reviewed.

## Real treasury funding

- [ ] Treasury Safe approval.
- [ ] Source of funds documented.
- [ ] Accounting treatment reviewed.
- [ ] Treasury risk limits reviewed.
- [ ] Incident response rehearsal completed.

## Staking or rewards

- [ ] Security review.
- [ ] Legal review.
- [ ] Tax/accounting review.
- [ ] Smart-contract audit if new code is involved.
- [ ] Governance approval.

## Buyback program

- [ ] Market-conduct legal review.
- [ ] Treasury policy review.
- [ ] Public disclosure plan.
- [ ] Governance approval.
- [ ] Monitoring plan.

## Autonomous execution

- [ ] Not approved for v1.
- [ ] Requires separate audit.
- [ ] Requires separate governance approval.
- [ ] Requires emergency stop controls.
- [ ] Requires extensive monitoring.
