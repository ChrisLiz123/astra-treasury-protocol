# Event-Driven Maintenance

## Triggering events

- Incident detected.
- Public status change.
- Governance evidence update.
- Monitoring configuration change.
- Domain / public-site change.
- Repository release package update.
- Any proposed capability activation.

## Required response

- Run public refresh.
- Run evidence archive.
- Update incident summary if applicable.
- Run affected gates.
- Commit and publish updated public artifacts if material.

## Rule

Event-driven maintenance must not be used to bypass action-specific approval.
