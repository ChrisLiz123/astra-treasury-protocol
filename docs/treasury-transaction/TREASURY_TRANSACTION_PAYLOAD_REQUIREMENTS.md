# Treasury Transaction Payload Requirements

## Status

No payload prepared.

## Required before any future payload

- Funding source approved.
- Treasury risk limits approved.
- Treasury Safe approval package complete.
- Public disclosures approved.
- Governance approval recorded.
- Funding amount within approved limits.
- Destination address verified.
- Asset address verified.
- Calldata decoded and reviewed.
- Safe nonce reviewed.
- Safe signer availability confirmed.
- Incident response process current.

## Payload requirements

A future transaction payload must include:

- Network.
- Safe address.
- Asset.
- Amount.
- Destination.
- Purpose.
- Approval references.
- Decoded calldata.
- Risk-limit reference.
- Source-of-funds reference.
- Public disclosure reference.

## Rule

No payload should be generated until the required approvals are complete.
