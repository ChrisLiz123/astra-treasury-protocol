# Capability Request Review Boundaries

## Review can do

- Confirm a request is valid for review.
- Confirm the requested capability is known.
- Recommend opening a dedicated action-specific approval path.
- Keep public status clear that no capability is approved.

## Review cannot do

- Approve full launch.
- Approve treasury funding.
- Approve public sale.
- Approve staking or rewards.
- Approve buybacks.
- Generate a Safe payload.
- Enable execution queue.
- Move funds.
- Execute any transaction.
- Activate automation or autonomous execution.

## Rule

A dedicated action-specific path is still required after review.
