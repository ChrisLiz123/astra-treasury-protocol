# Capability Activation Boundaries

## Required for any future capability activation

- Explicit action-specific approval.
- Governance decision or capability-specific authority record.
- Evidence archive reference.
- Public status update.
- Mainnet monitor passing.
- Active incidents zero.
- Safe review if any on-chain action is involved.
- Dedicated activation gate.

## Capabilities requiring separate approval

- Full launch.
- Public token sale.
- Real treasury funding.
- Staking or rewards.
- Buyback program.
- Mainnet execution queue.
- Paper-to-on-chain automation.
- Autonomous execution.
- Safe payload generation.
- Safe transaction execution.

## Rule

The restricted-mode operations handoff does not approve any capability.
